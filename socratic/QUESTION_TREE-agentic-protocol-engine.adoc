== Q1: What problem does this bounded context solve, and for whom?

=== Q1.1: Product identity — what is this?
[ANSWERED]
Evidence: pyproject.toml:2 (name = "agentic-protocol-engine"), pyproject.toml:4 (description), README.md:1-6
A generic state machine for running reliable, multi-step *agentic processes on
GitHub*: you declare a protocol as data (phases, agent steps, deterministic
checks, human gates) and one engine drives it — plus a read-only visibility REST
API over the engine's state.

==== Q1.1.1: What is the durable unit of work?
[ANSWERED]
Evidence: CLAUDE.md:"A workflow run is one transition of a state machine whose state lives in git", lib.py:645-660 (state_checkout), lib.py:663 (cas_push)
One workflow run = one transition of a state machine whose state is a YAML file
on the `agentic-state` git branch; compute is ephemeral, state is durable.

==== Q1.1.2: What is the control-inversion claim vs. porch/gh-aw?
[ANSWERED]
Evidence: README.md:42-45, CLAUDE.md:"The engine drives; the agent is dispatched"
The engine drives and the agent is dispatched: the agent exists only when the
engine spawns it, can affect the world only through an `evidence.json` artifact,
and is gone before the engine writes state — it cannot skip the engine.

=== Q2.6: Cross-cutting business rules
==== Q2.6.StateSacred: State advances only by fast-forward CAS
[ANSWERED]
Evidence: lib.py:663-683 (cas_push, attempts=5, rebase retry, never force), CLAUDE.md ("Never force-push agentic-state")
Non-initial state is written only by fast-forward compare-and-swap; `agentic-state`

==== Q2.6.EngineAgentSeparation: Engine and agent never share a job/credential
[ANSWERED]
Evidence: CLAUDE.md (four trust zones invariant), agentic-engine.yml:42-46 (plan perms) vs :442-445 (checks read-only)
The load-bearing rule: the engine (state PAT) and the agent (read-only + LLM
creds) never share a job or credential; the checks job even re-fetches `gh pr diff`
itself rather than trusting agent data.
==== Q2.6.NoShellInjection: Agent strings via env, never interpolated
[ANSWERED]

Agent-derived strings (feedback, verdicts, comment bodies, filenames) are passed
to shell via `env:`, never interpolated into `run:` blocks.
==== Q2.6.EventsAreWakeups: Events carry no state
[ANSWERED]
Evidence: CLAUDE.md ("Events are wake-ups, not state carriers"), next.py:6-12 (commands re-derive from state)
A trigger only tells the engine to look; everything load-bearing is re-derived
from the state file, not from the event payload.
